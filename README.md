# FLORA — цветочный магазин

Next.js 16 + Tailwind + Prisma + PostgreSQL.

Проект в текущем виде сочетает:
- рабочий пользовательский флоу каталога/корзины/чекаута;
- квиз подбора с сохранением подборки;
- регистрацию/логин (локально, через `localStorage`);
- минимально рабочее оформление подписки.

## Быстрый старт

1. Установите зависимости:
```bash
npm install
```

2. Создайте `.env`:
```bash
cp .env.example .env
```

3. Укажите `DATABASE_URL` в `.env` (PostgreSQL).

4. Выполните миграции:
```bash
npx prisma migrate dev
```

5. Запустите dev-сервер:
```bash
npm run dev
```

## Что уже реализовано

### Каталог и покупка
- Главная: hero + блок популярных букетов.
- Каталог: фильтры по поводу и цене, сортировка.
- Карточка букета: галерея, состав, количество, добавление в корзину.
- Корзина: изменение количества, апсейл открытки.
- Чекаут: данные получателя, слот/дата, телефон с маской, открытка с шаблонами и превью.
- Заказы: `POST/GET /api/orders` с сохранением в PostgreSQL (через Prisma).

### Квиз подбора
- Квиз с шагами: повод, бюджет, цвет, формат.
- Страница результатов: подбор 3–5 букетов по ответам.
- Кнопка сохранения подборки в `localStorage` (`flora:last-quiz-selection`).
- Переход в каталог с уже примененными фильтрами.

### Аккаунт
- Регистрация: `/register`.
- Вход: `/login`.
- Личный кабинет: профиль, история заказов, мои поводы.
- Вкладка "Подписки" и блок "Последний подбор" (данные из `localStorage`).

### Подписка на букеты (MVP)
- Страница тарифов: `/subscription`.
- Оформление подписки: `/subscription/checkout?plan=...`.
- Страница успеха: `/subscription/success`.
- Заявки подписки сохраняются в `localStorage` (`flora_subscriptions`).
